About This Simulation

This interactive 3D Solar System simulation is a project developed to showcase advanced frontend development skills, specifically leveraging the power of Three.js, a robust JavaScript 3D library built on top of WebGL. The goal is to provide an engaging and visually appealing representation of our celestial neighborhood.

The simulation features:

  • Accurate representation of the Sun and the 8 major planets (Mercury, Venus, Earth, Mars, Jupiter, Saturn, Uranus, Neptune).
  • Real-time orbital and rotational animations.
  • Dynamic control over global simulation speed.
  • Individual speed adjustments for each planet's orbit.
  • Interactive camera controls (orbit, zoom, pan) for a personalized viewing experience.
  • Smooth camera focusing on clicked planets.
  • Informative planet labels on hover.
  • Responsive design for optimal viewing on various devices.
  • Optimized asset loading with a visual progress indicator.

All high-resolution textures for the celestial bodies and the background starfield are sourced from Solar System Scope, a fantastic resource for astronomical visualizations. Special thanks to their team for providing these assets.

How to Use This Simulation

Follow these simple instructions to navigate and interact with the solar system.

  • Camera Orbit: To rotate your view around the entire solar system, click and drag your primary mouse button (usually left-click) anywhere on the main simulation canvas. This allows you to observe the planets from different angles.
  • Camera Zoom: Use your mouse scroll wheel to zoom in and out of the solar system. Scrolling up will zoom in, bringing you closer to the celestial bodies, while scrolling down will zoom out, providing a wider perspective. On touch devices, use a two-finger pinch gesture.
  • Camera Pan: To move the camera laterally across the scene (without changing its orientation relative to the scene), hold down your secondary mouse button (usually right-click) and drag. This is useful for shifting your view to specific regions. On touch devices, use a two-finger drag.
  • Focus on Planet: For an immersive experience, simply click on any planet within the 3D simulation area. The camera will smoothly transition and focus its view directly on the selected planet, allowing for a closer inspection.
  • Global Speed Control: The "Global Speed Multiplier" slider in the controls panel allows you to adjust the overall pace of the simulation. Dragging the slider to the left (towards 0) will slow down all orbital and rotational movements, effectively pausing the simulation at 0x speed. Dragging to the right (towards 2) will accelerate all movements.
  • Individual Planet Speeds: Below the global speed control, you will find dedicated sliders for each of the eight planets. These sliders enable you to fine-tune the orbital and rotational speed of each specific planet independently, relative to the global speed setting. For example, you can make Earth orbit faster while Jupiter remains at normal speed.
  • Pause/Resume Simulation: The "Pause Simulation" / "Resume Simulation" button acts as a master toggle for all animations. Clicking it will halt all planetary movement, and clicking it again will resume the simulation from its current state.
  • Reset Camera View: If you get lost in space or simply wish to return to the default wide-angle view of the entire solar system, click the "Reset Camera View" button. The camera will smoothly transition back to its initial position and orientation.
  • Planet Labels: As you move your mouse cursor over a planet, its name will appear in a small label, providing quick identification. This feature leverages raycasting to detect intersections with celestial bodies.

Technical Details

This project is structured using standard web technologies: HTML for content, CSS for styling, and JavaScript for interactivity and 3D rendering.

  • HTML5: Semantic markup, responsive meta tags, and accessibility attributes (ARIA roles) are used to create a well-structured and accessible user interface.
  • CSS3: Modern CSS features, including custom properties (CSS variables), Flexbox for layout, `backdrop-filter` for frosted glass effects, and comprehensive media queries ensure a visually appealing and mobile-responsive design across various screen sizes.
  • JavaScript (ES6+): The core logic is written in modern JavaScript, utilizing classes for object-oriented structure (`SolarSystemSimulation`), `Promise` for asynchronous texture loading, and `requestAnimationFrame` for optimized animation loops.
  • Three.js (r128): A powerful and widely-used JavaScript 3D library that abstracts WebGL complexities. It's used for:
    • Scene management (Scene, Camera, Renderer).
    • Geometry creation (SphereGeometry, RingGeometry).
    • Material application (MeshStandardMaterial for planets, MeshBasicMaterial for Sun/Rings).
    • Texture mapping for realistic surface details.
    • Lighting (AmbientLight, PointLight).
    • Camera controls (OrbitControls).
    • Raycasting for interactive elements (hover, click).
  • Performance Optimization:
    • `requestAnimationFrame` ensures animations are synchronized with browser refreshes, leading to smoother visuals and better battery efficiency.
    • `THREE.Clock` is used to calculate `deltaTime`, making animations frame-rate independent.
    • `THREE.LoadingManager` provides a robust way to track and manage asset loading, allowing for a smooth transition from loading screen to simulation.
    • `enableDamping` in OrbitControls provides a more natural and smooth camera movement.

Troubleshooting

If you encounter any issues, these tips might help.

  • Black Screen / No Planets: This is often due to texture loading failures. Ensure all planet and star textures are downloaded to `assets/textures/` and that `script.js` uses relative paths (`./assets/textures/texture.jpg`).
  • CORS Errors: If you see "Cross-Origin Request Blocked" errors in your browser's console, it means textures cannot be loaded from external URLs directly. You MUST run this project using a local web server (e.g., Python's `http.server` or Node.js `serve`). Simply opening `index.html` from your file system (`file:///`) will not work due to browser security policies.
  • Performance Issues: If the simulation runs slowly, try closing other demanding applications or browser tabs. Ensure your browser and graphics drivers are up to date.
  • Controls Not Responding: Make sure JavaScript is enabled in your browser. Try refreshing the page.
  • Mobile Responsiveness: While designed to be responsive, extreme variations in device aspect ratios might cause minor layout quirks.
  • Browser Compatibility: This simulation relies on WebGL (via Three.js). Ensure your browser supports WebGL and that it's enabled. Most modern browsers support it by default.
